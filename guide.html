For Map Creators

1

Upload your CSV

Drag it into the left drop zone, or grab Download CSV template in the sidebar first. First 3 rows optionally set Title/Subtitle/Date:

RouteTitle,Bangladesh, Cox's Bazar
Subtitle,Donor field mission
VisitDate,2026-07-01
Sl,Time,Duration,Activity,Camp,Agency,lat,lon
2

Upload a logo (optional) — second drop zone next to CSV. Appears on the map and in exports. RCP logo is the built-in default if you skip this.

3

Export PNG/PDF — PDF shows a preview first; header/footer repeat on every page automatically. Files auto-name as subtitle_date.

4

Browser Link — Save Current → Share. Instant, but only works on the same device you saved it on.

5

Publish to GitHub (any device)

Share → GitHub settings → fill in username / repo / branch / folder + token → Save Settings. Then just click Publish to GitHub & Get Link each time.

Creating a GitHub token

  1. github.com → Settings → Developer settings → Fine-grained tokens
  2. Generate new token → name + expiration
  3. Repository access → "Only select repositories" → your repo
  4. Permissions → Repository box (not Account) → Contents → Read and write
  5. Generate & copy immediately — shown once only

Treat it like a password. Stored only in your browser.

For Visitors / Viewers

If someone sent you a route link — no account or login needed.

Opening the link

Tap it in email, WhatsApp, or Teams — opens straight in your browser, nothing to install.

Reading the map

Numbered markers = stops in order. Tap any marker for time, duration, camp, agency.

Live location

Tap Track My Location, allow the permission prompt — a blue dot shows where you are, updating live.

Needs GPS/location on. Accuracy depends on your device.

On mobile

Title collapses — tap the ▾ arrow to expand. Route summary + disclaimer live behind the round ⓘ button on the map.

Troubleshooting

"Failed to publish: Not Found"

Username, repo name, or branch doesn't match — or your token can't see that repo. Check Repository access on the token.

"Resource not accessible by token"

Token can see the repo but can't write. Add Contents: Read and write under the Repository permissions box (not Account), then regenerate.

Site shows README, not the map

GitHub Pages needs a file named exactly index.html. Rename your app file if needed.

PDF/PNG download does nothing

Confirm you're on the latest app.js. If it persists, open DevTools (F12) → Console for the red error message.

Token permission dropdown missing

Make sure you're adding permissions under the Repository section, not Account — they look similar but are separate boxes.
