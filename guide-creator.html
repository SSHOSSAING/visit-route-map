Overview

Visit Route Map lets you create interactive route maps from a simple CSV file. Upload your data, customize the appearance, and share or export professional-quality maps for field missions, site visits, travel itineraries, and more.

i
What you can do
Upload route data via CSV, add a custom logo, export as PNG or PDF, share via browser link, or publish to GitHub for permanent sharing.
★
No account required
Everything runs in your browser. No sign-up, no server, no data leaves your device unless you choose to publish to GitHub.

Quick Start

Follow these five steps to create your first route map. Each step builds on the previous one.

1

Upload your CSV file

Drag your CSV into the left drop zone, or click Download CSV template in the sidebar to get started. The first three rows can optionally set the map title, subtitle, and visit date.

RouteTitle,Bangladesh, Cox's Bazar
Subtitle,Donor field mission
VisitDate,2026-07-01
Sl,Time,Duration,Activity,Camp,Agency,lat,lon
i
CSV format
The header row must include Sl, Time, Duration, Activity, Camp, Agency, lat, and lon. Coordinates should be in decimal degrees.
2

Add a logo (optional)

Use the second drop zone next to the CSV uploader. Your logo appears on the map and in all exports. If you skip this, the default RCP logo is used automatically.

3

Preview and verify

The map renders automatically once your CSV is loaded. Check that markers appear in the correct order, pop-ups show the right data, and the route line connects stops logically.

4

Export or share

Use the export buttons for PNG or PDF, or save a browser link for quick sharing. See the sections below for detailed export and sharing options.

5

Publish for permanent access

For links that work on any device, publish to GitHub. This creates a permanent, shareable URL that anyone can open without needing your original file.

Exporting Maps

Export your route map as a high-quality image or print-ready PDF. Both formats include your title, subtitle, logo, and all route markers.

PNG Export

The PNG export captures the current map view as a high-resolution image. This is ideal for embedding in reports, presentations, or emails.

  • Click the Export PNG button in the toolbar
  • The file auto-names as subtitle_date.png
  • Resolution matches your current viewport size
★
Tip for high-resolution exports
Before exporting, zoom and pan to frame your route exactly as you want it. The PNG captures the visible map area.

PDF Export

The PDF export auto-compacts the map to fit on a single page. This is perfect for printing or attaching to formal documents.

  • Click the Export PDF button in the toolbar
  • The layout automatically scales to fit one A4 page
  • File auto-names as subtitle_date.pdf
!
PDF export not working?
Make sure you are using the latest version of app.js. If the issue persists, open DevTools (F12) → Console and check for error messages.

Sharing Routes

Share your route map with team members, donors, or stakeholders. Choose between a quick browser link or a permanent GitHub publish.

Browser Link

The fastest way to share — save the current map state to your browser's local storage and generate a shareable link.

  1. Click Save Current to store the route in your browser
  2. Click Share to copy the link to your clipboard
  3. Paste the link into email, WhatsApp, Teams, or any messaging app
!
Device limitation
Browser links only work on the same device and browser where you saved them. For cross-device sharing, use GitHub publish instead.

Publish to GitHub

Publish your route map to GitHub Pages for a permanent, device-independent shareable link. This is the recommended method for formal distribution.

  1. Click Share → GitHub Settings
  2. Enter your GitHub username, repository name, branch, and target folder
  3. Paste your personal access token (see token setup below)
  4. Click Save Settings
  5. Click Publish to GitHub & Get Link each time you want to publish
i
GitHub Pages requirement
Your repository must have GitHub Pages enabled. The published file must be named index.html for GitHub Pages to serve it correctly.

Creating a GitHub Token

A fine-grained personal access token is required for the app to write files to your repository.

  1. Go to github.com → Settings → Developer settings → Fine-grained tokens
  2. Click Generate new token
  3. Name your token and set an expiration date
  4. Under Repository access, select Only select repositories and choose your target repo
  5. Under Permissions → Repository (not Account), set Contents to Read and write
  6. Generate the token and copy it immediately — it is shown only once
⚠
Security warning
Treat your token like a password. It is stored only in your browser's local storage and is never sent to any server other than GitHub's API.
!
Troubleshooting: "Failed to publish: Not Found"
This usually means the username, repository name, or branch is incorrect. Double-check spelling and verify your token has access to the selected repository.
!
Troubleshooting: "Resource not accessible by token"
Your token can read the repo but cannot write. Add Contents: Read and write under the Repository permissions box (not Account), then regenerate the token.

Best Practices

Data Preparation

  • Always use the CSV template as your starting point to ensure correct column headers
  • Verify latitude and longitude values before uploading — swapped or invalid coordinates will place markers incorrectly
  • Keep activity descriptions concise for clean pop-up rendering
  • Use consistent date formatting (YYYY-MM-DD) in the VisitDate row

Logo Guidelines

  • Use a square or landscape PNG with transparent background for best results
  • Recommended size: 400×200 pixels or larger
  • Keep file size under 500KB for fast loading

Sharing Workflow

  • Use Browser Link for quick internal reviews on your own device
  • Use GitHub Publish for final distribution to external stakeholders
  • Always test your shared link on a different device before sending it out
★
Pro tip: Version your routes
When publishing to GitHub, use different folder names (e.g., route-v1, route-v2) to keep a history of route versions without overwriting previous publishes.
